FASE 0: El Entorno - ¿Dónde Escribo el Código Mágico?

FASE 1: Los Bloques de LEGO - Variables y Tipos de Datos

El código se construye con datos. Las variables son como cajas con etiquetas donde guardas esos datos. Python es dinámicamente tipado, significa que no tienes que declarar el tipo de dato antes; Python lo deduce como Sherlock Holmes.

Creando Variables (¡Es como nombrar a tu Tamagotchi!)

Usas el signo igual (=) para asignar un valor a un nombre.

nombre_heroe = "Iron Man"           # Texto (string)
edad_stark = 53                 # Número entero (integer)
prob_snap = 0.00000007       # Número decimal (float)
es_genio_millonario = True   # Verdadero o Falso (boolean)
debilidad = None               # Representa "nada" o vacío (NoneType)

# Puedes imprimir para ver qué hay dentro
print(nombre_heroe)
print(edad_stark)
print(es_genio_millonario)

Tipos de Datos Fundamentales (Las Gemas del Infinito de Python)

  • str (String): Texto. Siempre entre comillas (" " o ' '). Puedes concatenar (unir) con +. 
    saludo = "Hola"
nombre = "Padawan"
mensaje = saludo + ", " + nombre + "!"  # Resultado: "Hola, Padawan!"
print(mensaje)
  • int (Integer): Números enteros. Sin decimales. 42, -10, 0.
  • float (Float): Números con decimales. 3.14159, -9.81. ¡Cuidado con la precisión a veces! No son perfectamente exactos.
  • bool (Boolean): Solo dos valores: True o False (¡con mayúscula inicial!). La base de toda lógica y decisiones.
  • NoneType: Representado por None. Significa ausencia de valor. Como el vacío antes del Big Bang.

Puedes chequear el tipo de una variable con la función type().

print(type(nombre_heroe))  # Output: <class 'str'>
print(type(edad_stark))      # Output: <class 'int'>
print(type(prob_snap))       # Output: <class 'float'>
Nombres de Variables Zen: Usa snake_case (minúsculas separadas por guion bajo), sé descriptivo pero conciso. numero_de_naves_rebeldes es mejor que n o num_naves. Haz que tu yo del futuro (o tu compañero de equipo) no te odie.

FASE 2: Las Herramientas - Operadores y Expresiones

Con los datos no basta, hay que *hacer* cosas con ellos. Los operadores son símbolos que realizan operaciones.

  • Aritméticos: Mates básicas. + (suma), - (resta), * (multiplicación), / (división real), // (división entera, descarta decimales), % (módulo, el resto de la división), ** (exponenciación). 
    puntos_vida = 100
dano_ataque = 35
puntos_vida_restantes = puntos_vida - dano_ataque # 65

nivel_poder = 9001
print("¡Es más de " + str(9000) + "!") # Necesitas str() para unir números a texto

print(10 // 3) # Output: 3 (División entera)
print(10 % 3)  # Output: 1 (Resto)
print(2 ** 10) # Output: 1024 (2 elevado a 10)
  • Comparación: Comparan valores y devuelven True o False. == (igual a), != (no igual a), > (mayor que), < (menor que), >= (mayor o igual que), <= (menor o igual que). 
    mi_edad = 30
edad_para_votar = 18
print(mi_edad >= edad_para_votar) # Output: True
print("Python" == "Java")        # Output: False
  • Lógicos: Combinan expresiones booleanas. and (verdadero si AMBOS son verdaderos), or (verdadero si AL MENOS UNO es verdadero), not (invierte el valor booleano). 
    tiene_entrada = True
es_mayor_de_edad = False
print(tiene_entrada and es_mayor_de_edad) # Output: False
print(tiene_entrada or es_mayor_de_edad)  # Output: True
print(not es_mayor_de_edad)           # Output: True

FASE 3: Hablando con el Mundo - Input y Output

Tu programa necesita comunicarse. Mostrar cosas (print) y pedir datos al usuario (input).

  • print(): Tu mejor amigo. Muestra lo que le pongas dentro en la consola. Puedes pasarle múltiples argumentos separados por coma. 
    planeta = "Tatooine"
soles = 2
print("Bienvenido a", planeta, "- Tenemos", soles, "soles!")
# F-Strings (Formato más moderno y legible!)
print(f"Bienvenido a {planeta} - ¡Tenemos {soles * 10} soles!") # Puedes poner expresiones dentro de {}
  • input(): Pausa el programa y espera que el usuario escriba algo y presione Enter. ¡Importante! Siempre devuelve el texto como un string, incluso si escribes números. Necesitas convertirlo si quieres hacer mates. 
    nombre_usuario = input("¿Cómo te llamas, viajero estelar? ")
print(f"Saludos, {nombre_usuario}!")

edad_texto = input("¿Cuántos años tienes? ")
# ¡ERROR si intentas sumar! edad_texto es string
# Necesitas convertir:
edad_numero = int(edad_texto) # Convierte a integer
print(f"En 10 años tendrás {edad_numero + 10} años.")
¡El Lado Oscuro de input()! Si pides un número con int(input()) y el usuario escribe "hola", tu programa explotará con un ValueError. Más adelante verás cómo manejar errores como un Jedi (try-except), pero por ahora, confía en que el usuario no es un Sith saboteador.

FASE 4: Agrupando Datos - Listas, Tuplas y Diccionarios

No siempre trabajas con datos sueltos. Necesitas formas de agruparlos. Python te da superpoderes con sus estructuras de datos.

Listas [] (Tu Mochila de Aventuras - Mutable)

Secuencias ordenadas de ítems. Puedes meter lo que quieras (números, strings, otras listas...). Son mutables: puedes cambiarlas después de crearlas (añadir, quitar, modificar).

equipo_avengers = ["Iron Man", "Capitán América", "Thor", "Hulk"]

# Acceder por índice (empieza en 0!)
print(equipo_avengers[0]) # Output: Iron Man
print(equipo_avengers[-1]) # Output: Hulk (índice negativo cuenta desde el final)

# Modificar un ítem
equipo_avengers[3] = "Profesor Hulk"
print(equipo_avengers)

# Añadir al final
equipo_avengers.append("Black Widow")
print(equipo_avengers)

# Quitar el último (o uno específico por índice)
heroe_caido = equipo_avengers.pop()
print(f"{heroe_caido} se fue...")
print(equipo_avengers)

# Longitud de la lista
print(len(equipo_avengers)) # Output: 4

Tuplas () (Coordenadas Estelares - Inmutable)

Similares a las listas, pero inmutables. Una vez creadas, no puedes cambiar sus elementos. Útiles para datos que no deben cambiar, como coordenadas o colores RGB.

coordenadas_base_rebelde = (42.0, -71.0, 1000.0)
print(coordenadas_base_rebelde[0]) # Acceso igual que listas

# ¡Esto daría ERROR! No puedes cambiar una tupla
# coordenadas_base_rebelde[0] = 10.0

Diccionarios {} (Tu Pokédex Personal - Pares Clave:Valor)

Colecciones no ordenadas (aunque en Python moderno mantienen el orden de inserción) de pares clave: valor. Accedes a los valores usando sus claves únicas (normalmente strings).

pokemon = {
    "nombre": "Pikachu",
    "tipo": "Eléctrico",
    "hp": 35,
    "ataques": ["Impactrueno", "Ataque Rápido"]
}

# Acceder a un valor por su clave
print(pokemon["nombre"])      # Output: Pikachu
print(pokemon["ataques"][0]) # Output: Impactrueno

# Añadir un nuevo par clave:valor
pokemon["debilidad"] = "Tierra"
print(pokemon)

# Modificar un valor existente
pokemon["hp"] = 40
print(pokemon["hp"])

FASE 5: Tomando Decisiones y Repitiendo - Control de Flujo

Un programa que solo va recto es aburrido. Necesitas tomar decisiones (if) y repetir tareas (loops).

Condicionales if / elif / else (Elige tu Propia Aventura)

Ejecutan bloques de código solo si se cumple una condición (booleana).

¡LA INDENTACIÓN ES LEY! Python usa espacios (normalmente 4) al inicio de la línea para definir bloques de código (lo que va dentro de un if, loop, función...). No uses tabuladores o mezcla de espacios y tabs. ¡La indentación incorrecta es el error más común y frustrante para principiantes! Es como olvidarte el punto y coma en otros lenguajes, pero peor. 
edad = 17
pais = "USA"

if edad >= 21:
    print("Puedes comprar alcohol (en USA).")
elif edad >= 18:
    print("Puedes votar (en USA).")
    if pais == "España": # Puedes anidar ifs
        print("Y también comprar alcohol (en España).")
else:
    print("Eres menor de edad para muchas cosas.")

print("Esta línea se ejecuta siempre, está fuera del if/elif/else.")

Bucles for (Recorriendo la Comarca)

Repiten un bloque de código para cada ítem de una secuencia (lista, tupla, string, etc.).

hobbits = ["Frodo", "Sam", "Merry", "Pippin"]

for hobbit in hobbits:
    print(f"{hobbit} está listo para la aventura!")

# Iterar un número específico de veces con range()
for i in range(5): # range(5) genera números del 0 al 4
    print(f"Contando ovejas eléctricas: {i}")

Bucles while (Esperando a Godot... o una Condición)

Repiten un bloque de código MIENTRAS una condición sea True.

¡Peligro de Bucle Infinito! Si la condición de un while nunca se vuelve False, el bucle se ejecutará eternamente (o hasta que detengas el programa a la fuerza - Ctrl+C). Asegúrate de que algo dentro del bucle pueda cambiar la condición eventualmente. ¡No crees un agujero negro computacional! 
bateria_restante = 10

while bateria_restante > 0:
    print(f"Nivel de batería: {bateria_restante}% - ¡Funcionando!")
    bateria_restante -= 1 # Decrementa la batería (¡IMPORTANTE para evitar bucle infinito!)

print("¡Batería agotada! Necesito recarga...")

FASE 6: Creando Herramientas Reutilizables - Funciones

Imagina que tienes un código que usas una y otra vez. Copiar y pegar es el camino al Lado Oscuro (y a los bugs). Las funciones te permiten empaquetar código, darle un nombre y reutilizarlo donde quieras. ¡Es como crear tus propios hechizos!

Siguen el principio DRY (Don't Repeat Yourself - No te repitas). Fundamental para código limpio y mantenible.

Definiendo y Llamando Funciones

Usas la palabra clave def, seguida del nombre de la función, paréntesis () (que pueden contener parámetros o argumentos de entrada), y dos puntos :. El código indentado debajo es el cuerpo de la función.

# Definir una función simple
def saludar():
    print("¡Hola, mundo desde una función!")

# Llamar (ejecutar) la función
saludar()

# Función con parámetros (entradas)
def saludar_a(nombre):
    print(f"¡Que la Fuerza te acompañe, {nombre}!")

saludar_a("Luke")
saludar_a("Leia")

# Función que devuelve un valor (salida) con 'return'
def sumar(num1, num2):
    resultado = num1 + num2
    return resultado # Devuelve el valor calculado

# Guardar el resultado de la función en una variable
total = sumar(10, 5)
print(f"El resultado de la suma es: {total}") # Output: 15

# Puedes usar el resultado directamente
print(f"Otro cálculo: {sumar(100, -20)}") # Output: 80
El Zen de las Funciones Puras: Intenta que tus funciones hagan UNA sola cosa bien. Idealmente, dadas las mismas entradas, siempre deberían producir la misma salida (determinismo) y no tener efectos secundarios (como modificar variables fuera de la función). Esto las hace más fáciles de testear y razonar. Piensa en ellas como mini-programas perfectos y autocontenidos.

FASE 7: El Código del Samurai - Buenas Prácticas (El Zen de Python)

Saber la sintaxis es una cosa. Escribir código elegante, legible y mantenible es otra. Es la diferencia entre un montón de ladrillos y una catedral.

  • PEP 8 - La Guía de Estilo Sagrada: Python tiene una guía de estilo oficial (PEP 8). Léela (o al menos las partes clave). Cosas como usar 4 espacios para indentar, líneas de máximo 79 caracteres, cómo nombrar variables y funciones (snake_case), clases (CamelCase), etc. VS Code con la extensión de Python te ayudará a seguirla. ¡Consistencia es clave!
  • Comenta con Propósito: Usa comentarios (# Esto es un comentario) para explicar el por qué de algo complejo o no obvio, no el qué hace el código (el código debería explicarse solo si es posible). 
    # MAL:
# Asigna 5 a x
x = 5

# BIEN:
# Usamos la constante de gravitación universal (aproximada)
GRAVITY_CONSTANT = 9.81
  • Nombres Descriptivos: Ya lo dijimos, pero es VITAL. tiempo_restante_segundos es infinitamente mejor que t.
  • Funciones Cortas y Enfocadas: Si una función hace demasiadas cosas, divídela.
  • Evita Números Mágicos y Strings Mágicos: No uses valores literales directamente en el código sin explicación. Guárdalos en variables con nombres descriptivos. 
    # MAL:
if status_code == 200:
    print("OK")

# BIEN:
HTTP_STATUS_OK = 200
if status_code == HTTP_STATUS_OK:
    print("Proceso completado exitosamente.")
  • Sé Consistente: Elige un estilo (idealmente PEP 8) y síguelo.
